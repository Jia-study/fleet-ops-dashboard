# 🚚 Fleet Operations Dashboard

> 쿠팡로지스틱스서비스(CLS) **Fleet Data & AI Operations Specialist** 프로토타입

차량 정비 데이터를 기반으로 Fleet 운영의 **가동률·비용·교체 시점**을 자동으로 분석하고, 룰 기반으로 즉시 조치가 필요한 차량을 탐지하는 대시보드입니다.

**🔗 Live Demo**: https://jia-study.github.io/fleet-ops-dashboard/

---

## 🎯 제작 배경

JD의 핵심 업무 4가지 (*차량 데이터 정리·AI 효율화·운영 인사이트·대시보드*) 를 한 화면에서 구현하여, **"데이터 → 인사이트 → 의사결정 지원"** 의 End-to-End 흐름을 보여주는 것을 목표로 했습니다.

---

## ✨ 주요 기능

### 1. 핵심 KPI 실시간 산출
- 전체 Fleet 가동률 / 운영 차량 수 / 12개월 총 정비비 / 긴급정비 비율

### 2. 🚨 이상 차량 자동 탐지 (룰 기반)
- **교체 권고**: 누적 정비비 300만원 초과 AND 연식 4년 이상
- **재정비 빈발**: 7일 내 2회 정비 AND 사후·긴급정비 포함
- **비용 이상**: 평균 대비 3배 이상 비용 기록

### 3. 💡 자동 생성 운영 인사이트
- 캠프별 긴급정비 비율 편차 탐지
- 정비소(벤더)별 평균 리드타임 비교
- 예방정비 비율 기반 정비 전략 제안
- 재정비 빈발 시 품질 QA 권고

### 4. 📊 시각화
- 월별 정비비 추이 (유형별 스택형)
- 정비 유형별 비율 (도넛 차트)
- 캠프별 평균 가동률 (수평 바 차트)

### 5. 🔍 차량별 검색·필터
- 차량번호·모델명 검색
- 배송캠프별·상태별 필터링

## 🛠 기술 스택

| 영역 | 사용 기술 |
|------|----------|
| Frontend | HTML · Vanilla JS · Pretendard |
| Visualization | Chart.js 4.4 |
| Data | JSON (가상 데이터 200건) |
| Hosting | GitHub Pages |

별도 빌드 툴 없이 **단일 HTML 파일 + JSON** 구조로 가볍게 구현하여, 운영팀이 로컬에서도 바로 사용할 수 있도록 설계했습니다.

---

## 📂 데이터 구성

- **차량 30대** (현대 포터2, 기아 봉고3 1톤 기준)
- **정비 기록 200건** (최근 12개월)
- **7개 배송캠프** (서울송파·강서, 경기부천·동탄, 인천서구, 대전대덕, 부산사상)
- **5개 정비소 벤더**
- **3개 정비 유형** (예방·사후·긴급)

> ⚠️ 공공데이터 포털의 차량 정비 API는 실명 기반 조회 구조로 Fleet 시뮬레이션에 부적합하여, 실제 물류 운영 맥락을 반영한 가상 데이터를 생성하여 활용했습니다.
